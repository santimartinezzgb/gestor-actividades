# ¿Directorio models?
El paquete `models` contiene las entidades del dominio que representan los datos persistidos o manejados por la aplicación.

Qué esperar:
- Clases que representan recursos (por ejemplo `User`, `Activity`, `Reserve`)
- Posibles enums (por ejemplo `RolUser`)
- Anotaciones de persistencia (por ejemplo `@Document` para Mongo)

Buenas prácticas:
- Mantener las entidades enfocadas en su estructura de datos
- Separar validaciones complejas en servicios o utilidades cuando sea necesario
- Evitar mezclar lógica de negocio compleja en las entidades si no es un patrón DDD explícito
