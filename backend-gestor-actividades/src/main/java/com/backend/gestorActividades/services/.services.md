# ¿Directorio services?
El paquete `services` contiene la lógica de negocio (servicios) que orquesta operaciones entre repositorios y controladores.

Qué verás:
- Servicios que encapsulan reglas del dominio (por ejemplo `AuthService`, `UserService`, `ActivityService`, `ReserveService`)
- Inyección de `repositories` y posibles utilidades o componentes externos

Buenas prácticas:
- Mantener responsabilidades claras (un servicio por contexto o agregado)
- Escribir tests unitarios para lógica compleja dentro de los servicios
- Evitar acoplar servicios a detalles de la capa web (mantenerlos reutilizables)
