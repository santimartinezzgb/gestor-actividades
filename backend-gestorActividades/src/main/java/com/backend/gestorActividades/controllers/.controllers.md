# ¿Directorio controllers?
El paquete `controllers` contiene los controladores REST que exponen la API de la aplicación. Su responsabilidad principal es manejar las solicitudes HTTP, validar/parsear inputs y delegar la lógica al servicio correspondiente.

Qué esperar:
- Endpoints agrupados por recurso (AuthController, UserController, ActivityController, ReserveController)
- Validaciones de entrada mínimas, transformación a DTOs y manejo de respuestas HTTP
- No debe contener lógica de negocio compleja (esa debe estar en `services`)

Buenas prácticas:
- Mantener controllers delgados (thin controllers)
- Usar DTOs para entrada/salida
- Manejar excepciones a través de un `@ControllerAdvice` centralizado (por ejemplo `GlobalExceptionHandler`)
